1) USB-to-UART driver (CP210x)

Most ESP32 dev boards expose USB via a USB‚Äëto‚ÄëUART bridge chip, commonly the Silicon Labs CP2102/CP210x. If the corresponding driver is missing, your board may not show up as a COM port.

Check in Windows

  1. Open Device Manager with Win + X, then M.
  2. Plug in the ESP32 with a Micro‚ÄëUSB cable.
  3. Look for a new COM port (for example under ‚ÄúPorts (COM & LPT)‚Äù).

If no new port appears, install the Silicon Labs VCP driver:

After installation, unplug/replug the board and verify the COM port appears.

For a quick prototype, we‚Äôll use MicroPython with the Thonny IDE. MicroPython is interpreted (great for rapid iteration); for performance‚Äëheavy applications, a compiled environment (e.g., Arduino/ESP-IDF) may be preferable.

  1. Install Thonny, then connect the ESP32.

  3. Select MicroPython (ESP32) and your board‚Äôs COM port.

3) Wiring the LED

Connect an LED to GPIO 16 through a 390 Œ© series resistor to ground:

  • GPIO16 ‚Üí resistor ‚Üí LED anode (+)
  • LED cathode (‚àí) ‚Üí GND

Tip: Keep the USB cable connected during development; the board can be powered over USB for this test.

4) The blinking script

Paste the following script into Thonny:

from machine import Pin
import time

# LED on GPIO 16 (adjust if you use a different pin)
led = Pin(16, Pin.OUT)

while True:
    led.value(1)
    print("1")
    time.sleep(0.5)

    led.value(0)
    print("0")
    time.sleep(0.5)

5) Run and validate

Click the Play button in Thonny to run the script from your computer.

If everything is set up correctly, the LED on GPIO16 will blink every 0.5 seconds, confirming the mockup is working.
